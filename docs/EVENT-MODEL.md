# Event Model

> Event naming, envelope, and NATS JetStream configuration for LOOM.

## Subject Hierarchy

All LOOM events are published to NATS JetStream using a hierarchical subject scheme:

```
loom.{tenant_id}.{domain}.{action}
```

### Domains

| Domain | Description |
|--------|-------------|
| `device` | Physical or virtual compute devices (servers, switches, firewalls) |
| `endpoint` | Network endpoints (IPs, ports, interfaces) |
| `workflow` | Workflow lifecycle events |
| `discovery` | Discovery scan results and reconciliation |
| `verification` | Verification check results |
| `audit` | Audit trail events (mirrored from audit records) |
| `cost` | Cost tracking and budget events |
| `lifecycle` | Asset lifecycle state transitions |

### Actions

| Action | Meaning |
|--------|---------|
| `created` | A new resource was created |
| `updated` | An existing resource was modified |
| `deleted` | A resource was removed |
| `discovered` | A resource was found during discovery |
| `enriched` | Additional facts were attached to a resource |
| `started` | A workflow or process began execution |
| `completed` | A workflow or process finished successfully |
| `failed` | A workflow or process finished with errors |
| `approved` | An approval gate was passed |
| `rejected` | An approval gate was denied |
| `conflict_detected` | Discovered state conflicts with known state |

### Subject Examples

```
loom.tenant-abc.device.created
loom.tenant-abc.workflow.started
loom.tenant-abc.discovery.conflict_detected
loom.tenant-abc.verification.completed
loom.tenant-abc.cost.updated
loom.*.workflow.failed          -- subscribe to all tenant workflow failures
loom.tenant-abc.>              -- subscribe to all events for a tenant
```

## Event Envelope

Every event published to NATS uses this envelope:

```go
package event

import "time"

// Event is the standard envelope for all LOOM events.
type Event struct {
    ID            string    `json:"id"`              // UUID v7 (time-sortable)
    Version       int       `json:"version"`         // Envelope schema version (start at 1)
    Timestamp     time.Time `json:"timestamp"`       // when the event occurred
    TenantID      string    `json:"tenant_id"`       // tenant scope
    Actor         Actor     `json:"actor"`           // who or what caused this event
    Action        string    `json:"action"`          // the action that occurred
    ResourceType  string    `json:"resource_type"`   // type of resource affected
    ResourceID    string    `json:"resource_id"`     // ID of the resource affected
    Data          any       `json:"data"`            // typed payload specific to the event
    CorrelationID string    `json:"correlation_id"`  // links all events in one request chain
    CausationID   string    `json:"causation_id"`    // links to the parent event that caused this one
}

// Actor identifies who or what caused an event.
type Actor struct {
    Type string `json:"type"` // "user", "system", "workflow", "adapter"
    ID   string `json:"id"`  // JWT subject for users, workflow ID for workflows, adapter name for adapters
}
```

### Field Rules

- **ID**: UUID v7 format. Generated by the publisher at publish time. Time-sortable for efficient querying.
- **Version**: Integer starting at 1. Represents the envelope schema version. Consumers MUST handle unknown versions gracefully (log warning + skip). See [WIRE-PROTOCOL-VERSIONING.md](WIRE-PROTOCOL-VERSIONING.md) Section 2 for evolution rules.
- **Timestamp**: UTC. Set by the publisher. Represents when the event actually occurred, not when it was published (these may differ by milliseconds under load).
- **TenantID**: Required. Every event is scoped to a tenant. System-level events use a reserved tenant ID (`_system`).
- **Actor**: Required. Always present, never empty. For system-initiated actions, `Actor.Type` = `"system"` and `Actor.ID` = `"{component}:{workflow_id}"`.
- **Action**: Required. One of the defined actions above. No freeform strings.
- **ResourceType / ResourceID**: Required. Identifies the resource this event is about.
- **Data**: Required for most events. Contains the event-type-specific payload. Must be a well-known struct, not `map[string]any` (except for adapter-specific metadata).
- **CorrelationID**: Required. Generated at the API boundary when a request enters LOOM. Propagated through all downstream events.
- **CausationID**: Required for derived events. For root events (triggered directly by user action), this equals the event's own ID. For events caused by another event (e.g., a workflow step completing triggers a device update), this is the ID of the causing event.

## Typed Event Payloads

Each domain/action combination has a specific payload type in the `Data` field:

```go
// DeviceCreatedData is the payload for device.created events.
type DeviceCreatedData struct {
    Hostname    string            `json:"hostname"`
    DeviceType  string            `json:"device_type"`
    ManagementIP string           `json:"management_ip"`
    Location    string            `json:"location"`
    Tags        map[string]string `json:"tags"`
}

// WorkflowStartedData is the payload for workflow.started events.
type WorkflowStartedData struct {
    WorkflowType    string `json:"workflow_type"`
    TargetResources []ResourceRef `json:"target_resources"`
    DryRun          bool   `json:"dry_run"`
}

// WorkflowCompletedData is the payload for workflow.completed events.
type WorkflowCompletedData struct {
    WorkflowType string `json:"workflow_type"`
    Converged    bool   `json:"converged"`
    Duration     string `json:"duration"`
    StepsRun     int    `json:"steps_run"`
}

// WorkflowFailedData is the payload for workflow.failed events.
type WorkflowFailedData struct {
    WorkflowType string `json:"workflow_type"`
    FailedStep   string `json:"failed_step"`
    ErrorCode    string `json:"error_code"`
    ErrorMessage string `json:"error_message"`
    Compensated  bool   `json:"compensated"`
}

// DiscoveryConflictData is the payload for discovery.conflict_detected events.
type DiscoveryConflictData struct {
    ResourceType string `json:"resource_type"`
    ResourceID   string `json:"resource_id"`
    Field        string `json:"field"`
    KnownValue   any    `json:"known_value"`
    ObservedValue any   `json:"observed_value"`
    Source        string `json:"source"` // which adapter observed this
}

// VerificationCompletedData is the payload for verification.completed events.
type VerificationCompletedData struct {
    WorkflowID  string `json:"workflow_id"`
    Converged   bool   `json:"converged"`
    TotalChecks int    `json:"total_checks"`
    Passed      int    `json:"passed"`
    Failed      int    `json:"failed"`
}

// ResourceRef used in event payloads.
type ResourceRef struct {
    Type string `json:"type"`
    ID   string `json:"id"`
}
```

## Event Guarantees

- Events are **append-only**. Once published, an event is never modified or deleted.
- Events are **immutable**. If a correction is needed, publish a new event (e.g., `updated` following a `created`).
- Events are published **at-least-once**. Consumers must be idempotent. Deduplication is by event ID.
- Event ordering is guaranteed **per subject** (NATS JetStream provides this). Cross-subject ordering is not guaranteed.

## NATS JetStream Stream Configuration

```go
// StreamConfig defines the NATS JetStream stream for LOOM events.
var LoomEventStream = &nats.StreamConfig{
    Name:        "LOOM_EVENTS",
    Description: "All LOOM domain events",
    Subjects:    []string{"loom.>"},
    Retention:   nats.LimitsPolicy,     // retain by limits (age + size)
    MaxAge:      90 * 24 * time.Hour,   // 90 days retention
    MaxBytes:    100 * 1024 * 1024 * 1024, // 100 GB default (see sizing note below)
    Storage:     nats.FileStorage,       // persist to disk
    Replicas:    3,                       // 3-way replication for durability
    Discard:     nats.DiscardOld,        // discard oldest when limits hit
    Duplicates:  5 * time.Minute,        // dedup window
    MaxMsgSize:  1 * 1024 * 1024,        // 1 MB max message size
}
```

**MaxBytes Sizing:** MaxBytes should be calculated as: `estimated_events_per_day x avg_event_size x retention_days x 1.5 safety_margin`. The default of 100GB is appropriate for deployments up to 10K devices with 90-day retention. Operators should monitor stream usage (`nats stream info LOOM_EVENTS`) and adjust MaxBytes before the stream approaches capacity, since `DiscardOld` will silently drop events that have not yet reached MaxAge.

### Consumer Patterns

| Consumer | Filter | Deliver | Use Case |
|----------|--------|---------|----------|
| DB Projector | `loom.>` | All | Project events into PostgreSQL for queries |
| Audit Writer | `loom.*.audit.>` | All | Write audit records to immutable store |
| Workflow Reactor | `loom.*.device.>`, `loom.*.endpoint.>` | New | Trigger reactive workflows on state changes |
| Alert Engine | `loom.*.workflow.failed`, `loom.*.verification.failed` | New | Send alerts on failures |
| Cost Tracker | `loom.*.cost.>` | All | Aggregate cost data |
| Tenant Notifier | `loom.{tenant_id}.>` | New | Per-tenant webhook/notification delivery |

### Consumer Configuration

```go
// Consumer config for the DB projector (must process every event).
var DBProjectorConsumer = &nats.ConsumerConfig{
    Durable:       "db-projector",
    DeliverPolicy: nats.DeliverAllPolicy,
    AckPolicy:     nats.AckExplicitPolicy,
    AckWait:       30 * time.Second,
    MaxDeliver:    5,                       // max redeliveries before dead-letter
    FilterSubject: "loom.>",
    MaxAckPending: 1000,
}
```

## Publishing Rules

1. Events must be published **after** the fact, not before. The thing must have happened before you announce it.
2. Events must include all data needed by consumers. Consumers should not need to call back to the API to process an event.
3. Events must not contain secrets. No passwords, tokens, or private keys in event data.
4. Large payloads (>100KB) should store the data externally and include a reference in the event.
5. Publishers must set the `Nats-Msg-Id` header to the event ID for deduplication.
